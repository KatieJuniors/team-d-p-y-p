RECEIPTS MODULE (INCOMING GOODS)

Receipts represent goods arriving from vendors.

Workflow:

1. Create receipt
2. Add supplier
3. Add products
4. Enter quantities
5. Validate receipt

When validated:

Stock automatically increases.

Example:

Receive 50 units of Steel Rods

Stock change:
+50 units

Receipt fields:

• Receipt ID
• Supplier
• Date
• Products
• Quantity
• Warehouse
• Status

Statuses:

Draft
Waiting
Ready
Done
Canceled

-----------------------------------------------------

DELIVERY ORDERS MODULE (OUTGOING GOODS)

Represents stock leaving the warehouse.

Workflow:

1. Create delivery order
2. Select products
3. Pick items
4. Pack items
5. Validate order

When validated:

Stock decreases automatically.

Example:

Delivery order for 10 chairs

Stock change:
-10 units

-----------------------------------------------------

INTERNAL TRANSFERS

Move stock inside the organization.

Examples:

• Main Warehouse → Production Floor
• Rack A → Rack B
• Warehouse 1 → Warehouse 2

Workflow:

1. Create transfer
2. Select source location
3. Select destination location
4. Add products
5. Confirm transfer

Rules:

• Total stock remains same
• Only location changes

-----------------------------------------------------

INVENTORY ADJUSTMENT MODULE

Used when physical stock does not match recorded stock.

Workflow:

1. Select product
2. Select location
3. Enter counted quantity
4. System calculates difference
5. Adjustment recorded

Example:

Recorded: 100
Actual: 97

Adjustment:
-3
